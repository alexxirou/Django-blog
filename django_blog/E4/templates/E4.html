Situation pofessionnelle 1: Mise en place d'une solution VPN en TLS

La société TEL-O qui offre des services support cliente via téléphone et en ligne souhaite mettre en place une connexion VPN pour que les développeurs soient capables de se connecter sur le serveur de développment à distance. La machine hôte en distance utilise Windows 10 et établir une connexion VPN via Internet vers le serveur de développment qui utilise l’OS Linux CentOS 8 en utilisant la solution OpenVPN pour se connecter sur le port 1194 UDP en utilisant le protocole TLS.

INSTALLATION D’UN SERVEUR LOG accessible via HTTPS

La société TEL-O qui offre des services support cliente via téléphone et en ligne souhaite mettre en place un serveur ‘journaux d’événements’ pour que le DSI soit capable de vérifier les événements et les erreurs de tous les ordinateurs connectés au réseau. Le serveur log est un IBM Powersystem E950 ou les services sont virtualisées sur des VMs utilisant le logiciel de virtualisation Promox. La connexion utilisée est 1 GO fibre dédiée par le FAI OVH. La solution retenue est d’utiliser rsyslog avec Adiscon Log-analyzer en conjonction avec la Base de Données MariaDB et le serveur HTTP Apache sur un OS CentOS 8. La machine hôte utiliser l'OS Windows 10 et utiliser le logiciel SolarWinds Event Forwarder pour transférer les journaux d’événements au serveur.
